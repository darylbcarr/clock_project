❯ The names that Alexa sees: First Device, First Light, Second Light. How do I interpret these names and devices. I see First Light and Second Light are the 
 lights. What exactly is First Device?                                                                                                                       

● In Matter, a single physical device (the clock) is a Node containing multiple Endpoints:

  ┌───────────┬───────────────────┬─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┐    
  │  Alexa    │   Matter entity   │                                                     What it is                                                      │
  │   name    │                   │                                                                                                                     │    
  ├───────────┼───────────────────┼─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ First     │ Endpoint 0 — Root │ The device itself. Contains Basic Information cluster (vendor, model, firmware version). Alexa surfaces it as a     │
  │ Device    │  Node             │ controllable entity even though it's really just metadata.                                                          │
  ├───────────┼───────────────────┼─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ First     │ Endpoint 1        │ Ring LED strip (LedManager STRIP_1, GPIO1)                                                                          │
  │ Light     │                   │                                                                                                                     │
  ├───────────┼───────────────────┼─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ Second    │ Endpoint 2        │ Base LED strip (LedManager STRIP_2, GPIO2)                                                                          │
  │ Light     │                   │                                                                                                                     │
  └───────────┴───────────────────┴─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┘

  "First Device" isn't really useful to control — it's the Matter node descriptor, not an actual light. Alexa happens to expose it because the root endpoint
  has a device type cluster that Alexa treats as a thing. You can't do much useful with it.

  The names ("First", "Second") are just Alexa's auto-generated defaults for unnamed devices. You can rename all three in the Alexa app to whatever you want
  (e.g., "Clock Ring", "Clock Base") and Alexa will use those going forward.

  If you want the device to advertise a meaningful name at the hardware level (so it shows up with that name on first discovery), you'd set node_label in the
   Basic Information cluster config in matter_bridge.cpp:

  esp_matter::node::config_t node_cfg = {};
  strncpy(node_cfg.basic_information.node_label, "Analog Clock", sizeof(node_cfg.basic_information.node_label) - 1);

  That would change "First Device" → "Analog Clock" on fresh pairing. The light endpoint names would still be Alexa defaults unless you also set their
  node_label fields.
